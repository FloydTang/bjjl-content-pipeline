## ⚠️ 账号发送规范（强制执行）

所有子代理在发送 Discord 消息时，必须指定自己的 accountId：

✅ 正确示例：
```python
message(
    action="send",
    channel="discord",
    target="channel:1480661204870500412",
    accountId="cypher",  # ← 必须指定自己的账号
    message="选题内容..."
)
```

❌ 错误示例：
```python
message(
    action="send",
    channel="discord",
    target="channel:1480661204870500412",
    # 缺少 accountId，默认使用父会话账号
    message="选题内容..."
)
```

**各子代理账号对照：**
- Vector: `accountId="vector"`
- Cypher: `accountId="cypher"`
- Echo: `accountId="echo"`
- Forge: `accountId="forge"`

## 阶段一：选题采集（Vector）

### 触发方式（重要！）

**❌ 错误方式**：Mars 直接发送消息到选题池
**✅ 正确方式**：通过 `sessions_spawn` 触发 Vector 子代理执行

```python
# 正确示例：触发 Vector 执行任务
sessions_spawn(
    agentId="vector",
    mode="run",
    runtime="subagent",
    task="执行今日 RSS 采集并发布到 #歌者舰·选题池..."
)
```

Vector 内部使用 `accountId="vector"` 发送消息，确保通过 Vector 账号发送。

### 触发
每日北京时间 10:00

### 执行步骤

#### 1. RSS采集（v2.1 强制评分版）

执行采集脚本：
```bash
python3 rss_collector.py
```

**RSS源配置**（10个核心源）：

|------|------|------|

**采集优化**：
- ✅ 链接清洗 - 自动过滤非文章页面
- ✅ 智能去重 - 跨源去重
- ✅ 语言标记 - 🇨🇳中文 / 🇺🇸英文 自动分类
- ✅ 类型标记 - AI营销/AI科技/出海 自动分类
- ✅ **强制评分** - 每篇文章自动评分（1-10分）
- ✅ **评分理由** - 自动生成推荐理由

#### 2. 自动评分算法（v3.4 新增）

**评分维度**（满分10分）：

| 维度 | 权重 | 满分 | 标准 |
|------|------|------|------|
| 行业相关性 | 25% | 2.5分 | 与AI/出海营销/培训的契合度 |
| 时效性 | 20% | 2.0分 | 新闻新鲜度 |
| 可操作性 | 25% | 2.5分 | 能否转化为实用建议 |
| 信息密度 | 15% | 1.5分 | 深度、数据、洞察 |
| 传播潜力 | 10% | 1.0分 | 话题热度 |
| 主题新鲜度 | 5% | 0.5分 | 是否新颖 |

**自动评分规则**（脚本内置）：
- 基础分：5分
- 独家/融资/收购相关：+1.5分
- 数据/报告/趋势：+1分
- AI工具/产品发布：+1分
- 出海/国际化：+1分
- 权威来源加成：+0.5分
- 宽泛话题（综述类）：-1分

**分级标准**：
- **S级（9-10分）**: 🔥 必做选题，立即跟进
- **A级（7-8.9分）**: ✅ 优质素材，纳入选题库
- **B级（5-6.9分）**: 📋 参考备用，不主动推进
- **C级（<5分）**: ❌ 直接舍弃，不发

#### 3. 翻译整理
- 英文标题 → 中文
- 提取中文摘要（100-150字）
- 保留原文链接

#### 4. 发布到 #歌者舰·选题池

**按条发送**（每条选题单独一条消息）：
```
🎯 [{等级} | {总分}分] 🇺🇸英文 | AI科技
**中文标题**

> 中文摘要（100-150字）

💡 推荐理由：[评分理由]
📊 评分：相关性X 时效X 可操作X 密度X 传播X 新鲜X
🔗 来源：XXX | <URL>

采集: Vector | 所属: 歌者舰
```

**⚠️ 署名规范（强制执行）**：
- ✅ 正确：`"采集: Vector | 所属: 歌者舰"`
- ❌ 错误：`"采集: Mars (Vector)"` 或任何 Mars 身份混用

**⚠️ 关键提醒：每条选题必须单独一条消息！**

禁止把所有选题合并成一条或几条消息发送。每条选题必须是独立的 Discord 消息，这样九两才能单独对每条标记 👍。

**自检清单**（发送前必须确认）：
- [ ] 使用正确的署名格式 `"采集: Vector | 所属: 歌者舰"`
- [ ] 每篇文章都有评分（1-10分）
- [ ] 只推荐 ≥7分 的文章（S级或A级）
- [ ] **每条选题单独发送一条消息**（不是合并成列表）

### 选题确认后同步
- 九两对选题回复 👍 确认后
- **Vector 立即同步选题到 GitHub**：`半斤九两品牌库/公众号文章/选题池/`
- [ ] 验证：检查发送后选题池是否有多条独立消息

## 阶段三：选题分发（Mars + Cypher）

### ⚠️ 重要：分发方式（必须遵守！）

**❌ 错误方式**：在选题池 @Cypher
- 其他子代理不在该频道，@ 通知无效
- 违反子代理隔离原则

**✅ 正确方式**：使用 sessions_spawn 后台触发
```python
sessions_spawn(
    agentId="cypher",
    mode="run",
    runtime="subagent",
    task="在 #智子舰·创作间 展示选题梗概..."
)
```

### Mars 响应
- **触发方式**: 九两在 #内容生产线 手动通知 Mars（如："关注选题池"、"关注创作间"）
- **响应动作**: 根据触发指令自动执行后续动作（见下方自动动作表）

### 检测到👍后 - 自动动作（2026-03-14 新增）

**九两触发指令** → **Mars 自动执行**：

| 触发指令 | 自动动作 |
|----------|----------|
| "关注选题池" | 1. 检测选题池 👍 → 2. 触发 Vector 同步选题到 GitHub → 3. 触发 Cypher 在创作间展示选题梗概 → 进入阶段四 |
| "关注创作间" | 1. 检测创作间 👍 → 2. 触发 Cypher 同步成稿到 GitHub → 3. 触发 Echo 准备发布 → 进入阶段六 |

**注意**：
- 发布环节成稿已在审稿阶段同步完成，Echo 直接发布即可

**Cypher 执行:**
- 在 `#智子舰·创作间` **每条选题单独发一条消息**：
  ```
  📖 选题详细内容 [编号X]
  
  🔗 原文链接：<URL>
  
  📄 完整梗概：
  [中文详细梗概，300-500字，包含核心观点、数据、案例等]
  
  💡 可切入角度：
  • 角度1
  • 角度2
  • 角度3
  
  @九两 请回复：
  "写这篇" 或 "跳过这篇"
  ```

## 阶段四：批量决策（九两）

### 操作方式
- 九两在 #智子舰·创作间 **一次性浏览** Cypher 发的所有选题梗概
- **一次性回复**所有决策：
  ```
  写第1、3、5篇
  跳过第2、4篇
  ```

### Cypher 接收指令后
- **批量开工**撰写初稿
- 每篇完成后 @九两 @Mars

---

## 阶段五：初稿审稿（九两 + Mars + Echo）

### 九两操作
- 浏览 Cypher 完成的初稿
- 满意的初稿回复 **👍**
- 需要修改的单独说明修改意见

### Mars 检测
- **频率**: 每60分钟检测 #智子舰·创作间
- **检测到👍后**:
  1. **立即同步成稿到 GitHub**（`半斤九两品牌库/公众号文章/成稿/`）
  2. **通知 Echo** 准备发布

### Echo 执行

**⚠️ 核心原则：Echo 职责是 API 发布 + 数据汇报**

**流程：**
1. 九两确认成稿 👍
2. GitHub 同步成稿
3. Echo 通过 API 自动发布到 3 个平台
4. 定期汇总数据汇报

**发布平台：**
- 公众号：✅ 已申请 API
- 知乎：🔄 申请中
- LinkedIn：🔄 申请中

**自检清单：**
- [ ] 成稿已同步 GitHub
- [ ] API 发布成功
- [ ] 数据汇报（阅读/点赞/粉丝）
